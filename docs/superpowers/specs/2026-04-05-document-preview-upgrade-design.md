# 文档预览能力升级设计（方案A）

## 目标

- 将文档列表中的“眼睛图标轻量预览”升级为“原文件级预览器”。
- 第一阶段以首开速度和稳定性优先：先秒开弹窗框架与首屏内容，再按需加载增强能力。
- 全链路保持本地执行，不新增云端预览服务上传路径。
- 为第二阶段“问答溯源联动预览”预留接口，避免返工。

## 范围（第一阶段）

- 统一预览弹窗框架（遮罩、顶部操作栏、主体预览区、底部状态栏）。
- 文件类型分发和独立渲染器：PDF、Excel/CSV、JSON、DOCX/TXT。
- 统一的本地预览资源加载层（按需加载、错误处理、资源释放）。
- 通用动作：关闭、下载、查看详情、定位分块（第一阶段固定行为：跳转详情页并携带 `documentId`，不做预览内高亮）。
- 统一异常兜底和降级路径。

## 非目标（第一阶段明确不做）

- 问答页点击溯源链接后直接在预览内精确高亮 chunk。
- PDF 高级批注、缩略图侧栏、复杂版面重排。
- Excel 公式引擎还原、复杂样式还原。
- JSON 高级全文检索跳转系统（跨层级高亮定位）。
- DOCX 版式级还原（先保证文本可读和稳定）。

## 现状与问题

- 当前预览逻辑位于 `frontend/src/pages/app/components/DocumentListPanel.tsx`，本质为 chunk 摘要展示，不是原文件预览。
- 预览能力与列表页耦合，后续扩展到多类型真实预览会导致状态和交互复杂度失控。
- 缺少统一预览数据入口，无法稳定支撑大文件懒加载、缓存释放和统一异常处理。

## 方案总览（推荐方案A）

采用“统一壳 + 类型适配器 + 懒加载渲染器”架构：

1. `PreviewModal` 负责通用框架和通用交互。
2. `DocumentPreviewContent` 负责按文件类型分发。
3. `PdfPreview` / `TablePreview` / `JsonPreview` / `TextPreview` 各自实现类型能力。
4. `useDocumentPreviewResource` 统一处理本地资源读取、加载态、错误态、释放。

该方案兼顾：

- 首开性能：预览器代码和数据按需加载。
- 可维护性：类型能力隔离，减少列表页复杂度。
- 可扩展性：第二阶段联动只需对接统一入口接口。

## 信息架构与交互规范

### 1) 弹窗结构

- 遮罩：半透明灰色，点击遮罩可关闭。
- 容器：居中显示，`max-width: 90%`、`max-height: 90%`、圆角统一 `8px`。
- 顶部栏：左侧文件名（超长截断+hover 显示完整），右侧按钮为“查看详情 / 下载 / 关闭”。
- 主体区：仅承载原文件预览，不混入 chunk 卡片信息。
- 底部栏：显示文件大小、类型、上传时间、分块总数；右侧“定位分块”按钮。

### 2) 交互原则

- 先打开弹窗骨架，再加载内容。
- 关闭弹窗后列表状态不变。
- 预览失败不影响下载与查看详情。

### 3) 视觉约束

- 主色语义令牌映射：`--preview-primary: #1677FF`（禁止在业务组件内硬编码，必须映射到现有主题 token）。
- 圆角令牌：`--preview-radius: 8px`。
- 风格：专业简约，减少非必要视觉噪声。
- 亮/暗主题均通过现有 token 系统派生，不引入独立主题分叉。

## 组件与职责边界

### 职责矩阵

| 模块 | 必须负责 | 禁止负责 |
|---|---|---|
| `DocumentListPanel` | 维护当前预览文档 ID、触发打开/关闭、承接查看详情回调 | 文件内容解析、类型渲染细节 |
| `PreviewModal` | 遮罩、Esc、头尾栏、通用按钮、布局尺寸 | 类型判定、文件读取 |
| `DocumentPreviewContent` | 类型判定与路由分发、渲染器加载占位 | 缓存策略、下载行为 |
| `useDocumentPreviewResource` | 请求与读取、缓存键、Abort、错误标准化、生命周期释放 | 具体 UI 展示 |
| `Pdf/Table/Json/TextPreview` | 各类型局部交互（翻页/展开/滚动）与渲染 | 通用弹窗动作与全局资源策略 |

下载编排归属：

- `PreviewModal` 只发出下载事件。
- `DocumentListPanel` 统一执行业务下载 use case。
- `useDocumentPreviewResource` 仅提供内容读取能力，不主动发起下载流程。

### `DocumentListPanel`（保留为容器）

- 管理“当前预览文档”状态。
- 触发 `openPreview(documentId, options?)`。
- 处理通用动作向上回调（查看详情、下载）。

### `PreviewModal`（新建）

- 负责遮罩、键盘 Esc、关闭、布局尺寸、头尾栏。
- 不关心具体文件类型渲染。

### `DocumentPreviewContent`（新建）

- 根据文档类型路由到具体渲染器：
  - `pdf` -> `PdfPreview`
  - `xlsx/xls/csv` -> `TablePreview`
  - `json` -> `JsonPreview`
  - `docx/txt` -> `TextPreview`
- 类型判定优先级：`MIME > 扩展名（lowercase）> fallback 文本预览`。

### `useDocumentPreviewResource`（新建 hook/service）

- 输入：文档 ID、类型、必要上下文。
- 输出：可渲染资源（如 objectURL、文本、JSON、表格片段）、加载态、错误态、释放函数。
- 统一处理：缓存策略、资源释放、异常封装。

### 各类型渲染器（新建）

- 只实现该类型内交互，不处理通用弹窗动作。

## 各文件类型预览策略

### PDF

- 第一阶段支持：翻页、缩放、页码跳转、基础文本选择。
- 采用首屏优先加载策略，后续页按需加载。
- 失败时降级到友好提示 + 下载 + 查看详情。

### Excel/CSV

- 结构化表格预览；Excel 支持 Sheet 切换，CSV 单表。
- 支持横纵滚动、表头固定。
- 大数据量时仅挂载首批行，后续按需追加。

### JSON

- 格式化展示、节点折叠/展开、复制。
- 默认展开浅层，深层按需展开。
- 超大 JSON 采用分段/分层加载，避免全量一次性渲染。

### DOCX/TXT

- 统一纯文本阅读模式。
- 支持滚动、基础搜索、快速复制。
- DOCX 首阶段优先稳定文本可读，不做复杂版式还原。

## 性能、稳定性与降级策略

### 三段式加载

1. 弹窗框架立即可见。
2. 类型渲染器与首屏内容按需加载。
3. 高级能力延后初始化（如更多页、更多行、深层节点）。

### 性能目标（第一阶段可量化指标）

- 首开骨架时间：`< 150ms (P95)`（本地已完成上传场景）。
- 首屏可读内容时间：`< 1.5s (P95)`（20MB PDF / 100k 行 CSV 基准样本）。
- 弹窗滚动与交互主线程卡顿：单次长任务 `<= 50ms`。
- 预览缓存上限：最近 `N=3` 个文档；超限按 LRU 回收。
- 关闭释放时机：弹窗关闭后 `3s` 内完成 objectURL、监听器和临时解析上下文释放。

### 资源管理

- 弹窗关闭时释放 objectURL、临时缓存、监听器。
- 可保留小规模近期预览缓存，但必须限制容量。
- 请求必须支持取消（`AbortController`），避免快速切换文档导致悬挂请求。

### 错误分级

| 错误码 | 场景 | 用户文案 | 可执行动作 | 是否可恢复 |
|---|---|---|---|---|
| `UNSUPPORTED_TYPE` | 类型不支持 | 当前文件类型暂不支持预览 | 下载、查看详情 | 否 |
| `NOT_FOUND` | 文件不存在/已删除 | 文件不存在或已被删除 | 刷新列表 | 否 |
| `PARSE_FAILED` | 文件损坏或解析失败 | 文件内容无法解析，请下载原文件检查 | 下载、查看详情 | 否 |
| `READ_FAILED` | 本地读取异常（IO/文件锁/瞬时失败） | 本地读取失败，请重试 | 重试加载、下载、查看详情 | 是 |
| `TOO_LARGE_PARTIAL` | 文件过大仅部分加载 | 文件较大，当前仅展示部分内容 | 继续加载、下载、查看详情 | 是 |
| `LOAD_TIMEOUT` | 本地读取超时 | 预览加载超时，请重试 | 重试加载、下载、查看详情 | 是 |
| `ABORTED` | 用户切换/关闭触发取消 | 已取消当前预览加载 | 重新打开 | 是 |

统一动作约束：错误态至少保留“下载 + 查看详情”；`重试加载` 与 `重新打开` 语义分离。

## 第二阶段联动预留（接口先行，功能后置）

第一阶段仅预留接口，不实现完整联动：

- `openPreview(documentId, options?)`
  - 预留参数：`chunkId`、`page`、`keyword`、`sheetName`、`jsonPath`
- `locateInPreview(target)`
  - 预留按类型定位能力入口
- `onOpenDetail(document, context?)`
  - 跳详情时携带上下文（未来用于落点定位）

第一阶段 `定位分块` 固定行为：

- 点击后执行 `onOpenDetail(document, { source: 'preview-locate' })`。
- 不在预览内执行 chunk 高亮与跳转。
- 若当前文档详情不可用，按钮置灰并显示提示文案。

## API 与类型契约建议

- 前端统一预览上下文类型（示意）：

```ts
type PreviewOpenOptions = {
  chunkId?: string;
  page?: number;
  keyword?: string;
  sheetName?: string;
  jsonPath?: string;
};
```

- 第一阶段可先在前端内部消费，不强制新增后端接口。
- 若需要原文件读取接口，应保证只访问本地存储，不新增云端代理链路。

### 预览资源 API 契约（建议落地）

- `GET /api/documents/:id/content`
  - 用途：预览读取与下载动作复用同一内容源，避免双实现。
  - 成功响应头：
    - `Content-Type`: 文件真实 MIME（或可推导 MIME）
    - `Content-Length`: 文件字节长度
    - `Accept-Ranges: bytes`（支持大文件分段读取）
  - 支持请求头：`Range`（按需读取首段/分页数据）

#### Range 协议最小契约

| 请求 | 响应 | 状态码 | 说明 | 前端动作 |
|---|---|---|---|---|
| 无 `Range` | 返回全量内容 | `200` | 常规读取（非分段） | 正常展示 |
| `Range: bytes=start-end` | 返回 `Content-Range` + 分段内容 | `206` | 分页/首段读取 | 正常展示并可继续请求后续段 |
| 非法或越界 `Range` | 返回 `Content-Range: bytes */total` | `416` | 请求范围无效 | 回退为首段请求或提示重试 |

补充要求：

- 必须返回 `Accept-Ranges: bytes`。
- `206` 响应必须返回 `Content-Range` 与本段 `Content-Length`。
- 第一阶段不支持多段 Range（multipart ranges），仅支持单段 bytes 范围。
- “部分预览可继续加载”必须通过 `206` 返回，禁止使用 `413` 表示可继续加载场景。
- 当服务端主动截断首段预览时，必须返回 `X-Preview-Partial: true`，用于前端展示“部分预览”提示。

- 状态码约定：
  - `200/206`: 正常返回内容（全量/分段）
  - `404`: 文档不存在或源文件缺失 -> 映射 `NOT_FOUND`
  - `413`: 请求不可处理（安全/策略拒绝），不作为可继续加载通道
  - `415`: 类型不支持 -> 映射 `UNSUPPORTED_TYPE`
  - `500`: 本地读取失败 -> 映射 `READ_FAILED`（默认可重试）

- 取消策略：
  - 前端对同文档重复预览请求进行去重。
  - 切换文档或关闭弹窗时，必须中止在途请求。

### 兼容约束

- 不改动现有上传、删除、重试、详情 API 的语义与返回结构。
- 仅新增预览相关接口或前端渲染层，不影响后台任务状态机。

### 统一错误映射

| HTTP/异常来源 | 统一错误码 | 默认文案 | 动作 |
|---|---|---|---|
| `404` | `NOT_FOUND` | 文件不存在或已被删除 | 刷新列表 |
| `206 + X-Preview-Partial: true` | `TOO_LARGE_PARTIAL` | 文件较大，当前仅展示部分内容 | 继续加载、下载、查看详情 |
| `413` | `READ_FAILED` | 当前请求无法处理，请重试或下载原文件 | 重试加载、下载、查看详情 |
| `415` | `UNSUPPORTED_TYPE` | 当前文件类型暂不支持预览 | 下载、查看详情 |
| 本地解析异常（可判定损坏） | `PARSE_FAILED` | 文件内容无法解析 | 下载、查看详情 |
| `500`/IO 异常/文件锁冲突 | `READ_FAILED` | 本地读取失败，请重试 | 重试加载、下载、查看详情 |
| 超时 | `LOAD_TIMEOUT` | 预览加载超时 | 重试加载、下载、查看详情 |
| Abort | `ABORTED` | 已取消当前预览加载 | 重新打开 |

### 错误响应 JSON 契约

非 `2xx` 响应统一返回：

```json
{
  "code": "READ_FAILED",
  "message": "本地读取失败，请重试",
  "retriable": true,
  "details": {
    "documentId": "doc_xxx",
    "reason": "FILE_LOCKED"
  }
}
```

约束：

- `code` 必须来自统一错误码集合。
- `retriable` 必须可机读，前端据此决定是否展示“重试加载”。
- `details` 为可选调试字段，禁止暴露敏感路径。

## 兼容迁移与发布策略

### 开关策略

- `ENABLE_NEW_PREVIEW_MODAL`：全局开关（默认开启）。
- `ENABLE_NEW_PREVIEW_BY_TYPE`：按类型开关（`pdf/table/json/text`）。

### 渐进发布顺序

1. 先启用新弹窗壳层与旧预览内容（验证壳层稳定性）。
2. 再逐步切换类型渲染器：`pdf -> table -> json -> text`。
3. 每切换一种类型后执行回归矩阵并观察错误率。

### 回退触发条件

- 预览首屏超时率（> 1.5s）超过 `5%`。
- 预览异常率超过 `3%`。
- 出现上传/删除/详情流程回归缺陷。

触发后：关闭对应类型开关，保留“下载 + 查看详情”路径。

### 观测口径（发布运行手册）

- 首屏超时率定义：`首屏时间 > 1.5s 的请求数 / 预览打开总请求数`。
- 异常率定义：`进入统一错误态的请求数 / 预览打开总请求数`。
- 统计窗口：5 分钟滑窗；同时观察 30 分钟累计。
- 统计维度：按类型拆分（pdf/table/json/text）+ 全局总览。
- 开关载体：运行时配置存储在后端本地配置源（`settings` 表或同等本地配置文件），前端通过 `GET /api/settings/preview-flags` 拉取。
- 执行主体：
  1. 自动守护任务 `preview-rollout-guard` 每 5 分钟计算阈值并写入类型开关。
  2. 若守护任务失败，值班同学需在 10 分钟内手动关闭对应类型开关。
- 触发动作：
  1. 关闭异常类型开关 `ENABLE_NEW_PREVIEW_BY_TYPE[type]=false`
  2. 保留其余类型继续运行
  3. 记录故障窗口并在 24 小时内复盘

## 测试与验收

### 单元与组件测试

- `PreviewModal`：遮罩关闭、Esc 关闭、按钮行为、头尾栏渲染。
- `DocumentPreviewContent`：类型路由正确性。
- 各渲染器：基础渲染、加载态、错误态、降级态。
- `useDocumentPreviewResource`：资源加载、释放、错误封装。

### 回归测试

- 列表页上传、删除、重试、查看详情流程不回归。
- 预览关闭后列表状态不变化。
- 国际化文案与主题（亮/暗）样式不回归。

### 性能验收协议

- 测试环境：
  - 机器：8C16G（或同等级）
  - 浏览器：Chrome Stable（固定主版本）
- 样本集：
  - PDF：20MB（300+页）
  - CSV：100k 行
  - JSON：50MB（深层嵌套）
  - TXT/DOCX：10MB 纯文本内容
- 每项重复次数：连续 10 次，统计 P50/P95。
- 采集方式：Performance API + 前端埋点（`preview_open_ts`、`modal_mount_ts`、`first_paint_ts`、`first_content_ts`、`error_code`）。
- 埋点触发点定义：
  - `preview_open_ts`：用户点击预览按钮时刻。
  - `modal_mount_ts`：`PreviewModal` 首次挂载完成时刻。
  - `first_paint_ts`：预览骨架首帧绘制完成时刻。
  - `first_content_ts`：首个可读内容节点可见时刻（PDF 首页可见/表格首行可见/JSON 根节点可见/文本首段可见）。
- 判定标准：满足本设计中的首开与首屏 SLA，且错误率阈值未超标。

### 关键回归场景矩阵

- 预览打开后，后台轮询刷新文档列表，弹窗不闪退且状态不丢失。
- 预览进行中删除当前文档，弹窗进入 `NOT_FOUND` 并保留可恢复动作。
- 文档失败后点击重试，再打开预览，状态可恢复且不影响详情跳转。
- 预览与详情页来回跳转后返回列表，筛选和滚动位置保持稳定。
- 切换语言/主题时，弹窗不重置当前页（或当前行/节点）并保持可交互。
- 快速连续打开不同文档时，前一个请求被取消且不会串内容。
- `Range` 协议一致性：`200/206/416` 路径可控且映射到预期 UI 动作。
- Abort 一致性：取消后不串内容、无悬挂请求。
- 资源回收验证：关闭弹窗后 3 秒内完成释放，重复开关不出现内存持续增长。
- 压力切换：连续 20 次切换不同文档仍保持可交互且不崩溃。

### 验收标准

- 各支持类型可打开原文件级预览（非 chunk 摘要）。
- 大文件场景满足首开 SLA：骨架 `P95 < 150ms`，首屏可读内容 `P95 < 1.5s`。
- 出现异常时可继续下载和跳详情，不出现“整弹窗不可用”。
- 视觉样式满足主色语义 token（映射自 `#1677FF`）、圆角 `8px`、统一布局规范。

## 风险与回退

### 主要风险

- 引入多类型渲染后组件边界不清，导致状态爆炸。
- 大文件渲染性能不足，影响主线程响应。
- 各类型错误处理不统一，用户体验割裂。

### 缓解策略

- 先落统一壳与分发层，再接入类型渲染器。
- 强制懒加载与分段渲染，不允许首版全量挂载。
- 统一错误模型和降级动作。

## 阶段边界约束

- Phase1 中 `openPreview(documentId, options?)` 仅 `documentId` 生效。
- `options` 中 `chunkId/page/keyword/sheetName/jsonPath` 仅透传与埋点记录，不驱动定位行为。
- 任何预览内定位、高亮、溯源闭环行为均属于 Phase2，禁止在 Phase1 实现中夹带。

### 回退策略

- 保留旧预览入口开关（必要时可快速切回轻量预览）。
- 新预览发布后若出现稳定性问题，先关闭类型渲染器增强路径，仅保留详情跳转与下载能力。
